Seguridad de Datos

Descripción completa de las medidas técnicas y organizativas que Argus implementa para proteger la información de denunciantes y organizaciones clientes.

Versión: 1.0 Última actualización: Febrero 2025 Alcance: Plataforma web y servicios Argus

Principios de seguridad

La arquitectura de seguridad de Argus se basa en tres principios fundamentales:

  • Privacidad por diseño (privacy by design): Las decisiones de diseño del sistema priorizan la protección de datos desde su concepción, no como una capa adicional.
  • Mínimo privilegio: Cada componente del sistema y cada persona tiene acceso únicamente a la información estrictamente necesaria para su función.
  • Defensa en profundidad: La seguridad se implementa en múltiples capas independientes, de modo que el fallo de una no compromete el sistema completo.

El sistema fue diseñado para que, incluso en el escenario de una brecha de seguridad, los datos de los denunciantes anónimos no puedan ser vinculados a personas identificables, dado que esa información nunca fue recopilada.

Capas de seguridad técnica

TLS 1.3

Cifrado en tránsito

Todas las comunicaciones entre el navegador del usuario y los servidores de Argus utilizan TLS 1.3, el protocolo de cifrado más moderno y seguro disponible.

AES-256

Cifrado en reposo

El contenido de todas las denuncias y datos sensibles se almacena cifrado con AES-256. Las claves de cifrado son gestionadas mediante un sistema de gestión de claves dedicado.

0 IP

Sin registro de IP

El sistema de denuncia no registra, procesa ni almacena la dirección IP del dispositivo utilizado. Esta decisión de diseño es irreversible a nivel de aplicación.

0 Cookies

Sin cookies de rastreo

La plataforma de denuncia no utiliza cookies de sesión persistentes, cookies de terceros ni tecnologías de seguimiento del comportamiento del usuario.

AISLADO

Entornos separados

Cada organización cliente opera en un entorno lógicamente aislado. Los datos de una organización no pueden ser accedidos por representantes de otra.

2FA

Autenticación fuerte

El personal de Argus y los representantes de organizaciones clientes utilizan autenticación de dos factores para acceder al panel de gestión.

Infraestructura y alojamiento

Servidores y centros de datos

La plataforma Argus opera en infraestructura con las siguientes características:

  • Centros de datos con certificación de seguridad física de nivel empresarial.
  • Redundancia geográfica para garantizar disponibilidad ante fallos regionales.
  • Entornos de producción, staging y desarrollo estrictamente separados.
  • Backups cifrados con retención conforme a las políticas de datos.

Residencia de datos

Por defecto, los datos se almacenan en servidores con redundancia regional. La residencia específica de datos puede configurarse por país según los requisitos regulatorios locales de cada organización cliente.

Proveedores de infraestructura

Los proveedores de infraestructura tecnológica tienen firmados acuerdos de confidencialidad y procesamiento de datos. No tienen acceso al contenido de las denuncias en texto plano.

Control de acceso

  • Principio de mínimo privilegio: Cada persona tiene acceso únicamente a la información necesaria para su rol.
  • Gestión de identidades: Los accesos se revocan inmediatamente ante cambios de rol o desvinculación.
  • Logs de auditoría: Todos los accesos a datos sensibles son registrados con timestamp, usuario y acción realizada.
  • Revisión periódica de accesos: Los permisos son revisados trimestralmente.
  • Separación de funciones: Ninguna persona tiene control unilateral sobre todos los sistemas críticos.

Desarrollo seguro

  • Revisión de código: Todo código que se incorpora a producción pasa por revisión de pares con foco en seguridad.
  • Testing de seguridad: El equipo realiza pruebas de seguridad periódicas, incluyendo análisis de vulnerabilidades y pruebas de penetración.
  • Actualizaciones: Las dependencias y componentes de software son actualizados con regularidad para incorporar parches de seguridad.
  • OWASP Top 10: El proceso de desarrollo contempla las vulnerabilidades más comunes definidas por OWASP.
  • Gestión de secretos: Las credenciales y claves nunca se almacenan en el código fuente; se gestionan mediante sistemas dedicados.

Respuesta ante incidentes de seguridad

Argus cuenta con un plan de respuesta ante incidentes de seguridad que incluye los siguientes pasos:

01

Detección y contención

Al detectarse un incidente, se activa el equipo de respuesta. La primera acción es contener el incidente para limitar su impacto.

02

Evaluación del impacto

Se evalúa qué datos pudieron haberse visto afectados, el alcance del incidente y si existe riesgo para los derechos y libertades de los titulares.

03

Notificación

En caso de brecha de datos con riesgo para las personas, se notifica a las organizaciones afectadas dentro de las 72 horas de confirmado el incidente. Si el riesgo es alto, también se notifica a los titulares afectados.

04

Remediación

Se implementan las correcciones necesarias para resolver el incidente y prevenir su recurrencia.

05

Análisis post-incidente

Una vez resuelto el incidente, se realiza un análisis de causa raíz y se actualiza el plan de respuesta según las lecciones aprendidas.

Capacitación y cultura de seguridad

  • Todo el personal recibe capacitación en seguridad de la información al incorporarse a Argus.
  • Capacitaciones periódicas sobre amenazas actuales, phishing y buenas prácticas.
  • El personal con acceso a datos sensibles recibe capacitación específica en protección de datos.
  • Existe un canal interno para reportar sospechas de incidentes de seguridad sin consecuencias para quien reporta.
